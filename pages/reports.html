Reports

Report Engine

Will these reports need to be printed? If so better to use a repot engine that can export to pdf and maintain line breaks for A4, otherwise SVG charts in browser could suffice.

By Individual PPP

Pop-up to select a Policy by title, and run report.

Date range that a notification was sent and valid to.

Can list vertically each school that has been sent any notification containing the selected policy. The status of response and compliance can also be shown.

Additional option to select multiple PPPs' with an 'and' or 'or' selector

An option may be to have dynamic colums to list response to chk list requirements ?

By Notification

Pop-up to select a sent notification alert.

Date range that a notification was sent and valid to.

Can list vertically each school that has been sent the selected notification and the response received.

An example of this report could be the results of a VRQA questionaire sent out for a given year.

An option to this report may be to select a specfic section and list compliance against the requirements of that section. (This level of detail will be available in the by School report)

By School

Pop-up to select a indvidual school.

Date range that a notification was sent and valid to.

Options to list vertically all notifications sent and status, or all PPP's in sent notificaitons and status.

Can select an individual notification and get a full break down each section and requirements and supporting documentation

  • SIMON submitted risk register, could be also available here

By PPP Category/Attribute

Pop-up to select a PPP Group, at any level

Date range that a notification was sent and valid to.

Aggregates policy doc notifications, showing most recent sent notification per attribute and if school has complied. Can list vertically each school and response status

Can select more than one group at a time with an 'and' option.
